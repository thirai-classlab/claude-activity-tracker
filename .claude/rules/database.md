## 環境変数のフォールバック（必須）

クライアント側・サーバー側で変数名プレフィックスが異なる場合はフォールバックを実装すること:

```typescript
const DB_URL = process.env.NEXT_PUBLIC_DB_URL ?? process.env.DB_URL;
```

**理由:** プラットフォーム統合によって片方しか注入されない場合があり、片方のみチェックすると本番環境でモードが切り替わってしまう。

**再発防止:** モード判定・クライアント初期化の両方でフォールバックを実装すること。
