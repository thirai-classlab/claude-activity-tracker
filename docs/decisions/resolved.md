## D-008: 1M tier 自動判定ロジック

- **起票日**: 2026-04-25
- **決定日**: 2026-04-25
- **当初決定**: A（usage 合計 > 200K で `-context-1m` variant 自動採用）
- **改訂日**: 2026-04-27
- **改訂理由**: usage 合計（cumulative cache reads 含む）は per-API-call の input サイズと無関係。同じ context を 1000 回読めば 200K を超え、誤って全セッション premium 課金になる。実測で全セッションが **1.96x の過剰課金** に。
- **改訂後の決定**: model 名に **`[1m]` literal suffix（Claude Code 由来）** または `-context-1m` を含む場合のみ premium tier を選択。usage-based 自動判定は撤回
- **影響**: DB 全セッションで再計算済み、`$858 → $438 (-$420.08)` に補正。ratio 1.96x ≈ 撤回前の過剰幅と一致
- **決定者**: ユーザー（当初） + Claude（改訂、観測値による自律判断）
- **関連**: `recalc-costs.ts` を `server/scripts/` に追加、idempotent
