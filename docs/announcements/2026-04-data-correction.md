# 【重要】ダッシュボード数値の訂正と修正対応について

> **パンくず**: [README.md](../../README.md) > [docs/](..) > [announcements/](.) > **2026-04-data-correction.md**
> **公開日**: 2026-04-25
> **対象**: Claude Code Activity Tracker 利用者全員

## 3 行まとめ

1. これまでダッシュボードに表示されていた**トークン数・コスト・ターン数・ツール利用数が最大 1.7〜62 倍に膨張していました**
2. 原因はトランスクリプトパーサの重複集計バグで、修正版をデプロイ済み（または順次デプロイ中）
3. **修正日より前のセッションには「計測不能期間」バナー**が表示されます。KPI 比較は修正日以降のデータで行ってください

## 何が起きていたか

Claude Code のトランスクリプト（会話ログ）は、1 回の API 応答内にテキスト＋ツール実行が混在する場合、同じ API 応答の行を複数書き込む仕様でした。当システムのパーサはこの重複を検出せずに合算していたため、以下が膨張していました。

| 項目 | 観測された膨張率 |
|------|-----------------|
| 総トークン数 / 入力・出力・キャッシュ | **1.7〜2.0 倍** |
| 推定コスト (USD) | **1.7〜2.0 倍** |
| セッション内ターン数 | **最大 62 倍**（例: 実 17 ターン → 表示 1059 ターン） |
| ツール利用数 | 2〜3 倍 |

例: セッション 18 の旧表示 `$370.20` は実際 `$190〜200` 相当でした。

## 影響範囲

- 2026-04-25 以前のすべてのセッションデータ
- 各ダッシュボード画面（Top / Members / Repos / Tokens / Sessions / Prompt Feed / Productivity）
- メンバー別・リポジトリ別 KPI

## 対応

### サーバ側

- `docs/specs/001-transcript-dedup.md` に基づくパーサ修正
- `server/src/services/transcriptParser.ts` / `setup/hooks/shared/utils.js` に `message.id` による重複排除を実装
- `handleStop` の `turn_count` / `tool_use_count` を DB 実数ベースに変更

### UI

- 修正日より前のセッション詳細画面に「🔔 計測不能期間」バナー表示
- Top 画面の合計 KPI には注意書き追記
- 新規データ（修正日以降）は正しい値で表示

### 過去データ

**再計算は行いません**（D-001 決定）。
- 各メンバー PC に残る生トランスクリプトから再集計することは技術的に可能ですが、運用コストと正確性のトレードオフから見送り
- 数週間のうちに修正後データが十分蓄積する見込み
- 過去データは「参考値」として残し、比較分析には新データを使用してください

## よくある質問

### Q1. 過去に「自分のコスト高すぎ！」と驚いたが、実は半分だったのか？

A. はい、おそらくそうです。直近のダッシュボードに切り替わった後の数値をご確認ください。

### Q2. 社内予算申請で過去の Tracker データを使ってしまった。訂正が必要？

A. 必要なら別途ご相談ください。月次予算の粒度であれば、実際の Anthropic 請求額（コンソール）と照合するのが確実です。

### Q3. 今後このような事態は？

A. 修正 PR に回帰テスト（CI）を同梱しており、同種のバグは二度と混入しません。また設計時点でレビューするよう `.claude/rules/autonomy.md` を整備しています。
